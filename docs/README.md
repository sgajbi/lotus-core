# Lotus Core Documentation

This directory is the documentation front door. Durable content belongs in the owned subtree that
matches its purpose; do not add uncategorized documents beside this index.

## Documentation Areas

1. [Architecture](./architecture/README.md) - current-state maps, decisions, review governance,
   and implementation evidence.
2. [Data](./data/database-schema-catalog.md) - database schema and data ownership reference.
3. [Features](./features/supported-features.md) - implementation-backed capability truth and
   feature documentation.
4. [Governance](./governance/api-governance.md) - API and security governance.
5. [Methodologies](./methodologies/) - calculation and analytical methodology.
6. [Operations](./operations/runbook.md) - runtime operations, observability, recovery, and
   incident support.
7. [RFCs](./RFCs/) - repository RFCs and implementation records.
8. [Standards](./standards/) - machine-readable and narrative engineering contracts.
9. [Testing](./testing/testing-strategy.md) - test strategy and validation guidance.
10. [Transaction Specifications](./rfc-transaction-specs/) - governed transaction behavior.

Individual codebase-review evidence records live under
[architecture/codebase-reviews](./architecture/codebase-reviews/README.md). Their current status
is indexed by the [codebase review ledger](./architecture/CODEBASE-REVIEW-LEDGER.md).
